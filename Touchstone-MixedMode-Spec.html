CONCEPT

1. Mixed-mode とは

差動信号（バランス信号）を扱う回路では、各信号ペアを「差動モード」と「コモンモード」の 2 つの独立した信号として解析すると便利です。 Mixed-mode Sパラメーター（別名 Balanced Sパラメーター）は、 シングルエンド（不平衡）測定で得た通常の Sパラメーター行列を 差動・コモンモード空間に変換したものです。

たとえば差動バッファや差動アンプ、差動伝送線路、バランなどの評価に使われます。 変換は線形演算なので測定値（シングルエンド S 行列）から計算で求められます。 専用のバランス対応 VNA は不要です。

  • 差動モード (D, Differential mode)：2 本の信号線が逆相で振幅が等しい状態
  • コモンモード (C, Common mode)：2 本の信号線が同相で振幅が等しい状態
THEORY

2. ポートモードの定義

差動モードとコモンモード

差動ペアのポートを p（正側, +）と n（負側, −）とします。 各ポートに入る電圧波 (a) と出る電圧波 (b) を用いると、 差動・コモンの入射波は次のように定義されます：

差動モード入射波： a_D = ( a_p − a_n ) / √2 コモンモード入射波： a_C = ( a_p + a_n ) / √2 同様に反射波： b_D = ( b_p − b_n ) / √2 b_C = ( b_p + b_n ) / √2

この 1/√2 の正規化係数はパワー整合（散乱行列のユニタリー性を保つ）のために必要です。 ソフトウェアや資料によっては 1/2 や 1/√2 の定義が混在しますが、 最も普及しているのは 1/√2 正規化です（IPC-2341A, Bockelman-Eisenstadt 定義）。

ポート番号の付け方

Mixed-mode 変換後の行列では、ポートをモードごとにグループ化します。 一般的な慣例では「差動ペア番号 + モード」でポートを命名します。

シングルエンド S 行列（変換前）
ポート 1: + 側（差動ペア 1, 正）
ポート 2: − 側（差動ペア 1, 負）
ポート 3: + 側（差動ペア 2, 正）
ポート 4: − 側（差動ペア 2, 負）
  ↓
  4×4 シングルエンド S 行列
Mixed-mode S 行列（変換後）
ポート 1: D1（差動ペア 1, 差動モード）
ポート 2: C1（差動ペア 1, コモンモード）
ポート 3: D2（差動ペア 2, 差動モード）
ポート 4: C2（差動ペア 2, コモンモード）
  ↓
  4×4 Mixed-mode S 行列
ℹ ポートの並び順は実装によって異なります。 「D1, D2, C1, C2」（差動を先にまとめる）や「D1, C1, D2, C2」（ペアごとにまとめる）など 複数の慣例があります。本ツールでは D1, C1, D2, C2 …（ペアごと）を採用しています。

3. 変換行列と計算式

モード変換行列 M

差動ペアが 1 組（シングルエンドポート 2 本 → Mixed-mode ポート 2 本）の場合、 変換行列 M は次の 2×2 行列です：

差動ペア 1 組の変換行列 M（2×2）： M = (1/√2) × [ 1 −1 ] ← 差動モード行 [ 1 1 ] ← コモンモード行 入力列ベクトル: [ a_p, a_n ]ᵀ → 出力: [ a_D, a_C ]ᵀ = M × [ a_p, a_n ]ᵀ

N ポートのうち複数ペアが混在する場合、M はブロック対角行列になります。 差動ペアが 2 組（4 ポート）なら：

4 ポート（差動ペア 2 組）の変換行列 M（4×4）： 列の順: [p1, n1, p2, n2] → 行の順: [D1, C1, D2, C2] [ 1 −1 0 0 ] M = (1/√2) × [ 1 1 0 0 ] [ 0 0 1 −1 ] [ 0 0 1 1 ]

シングルエンドポートが混在する場合、そのポートに対応する行・列は [ 1 ]（変換なし）をブロック対角に追加します。

Mixed-mode S 行列の計算

シングルエンド S 行列を S_se、 Mixed-mode S 行列を S_mm とすると：

S_mm = M × S_se × M⁻¹ M は正規直交行列なので M⁻¹ = Mᴴ（複素共役転置）です。 実装では M⁻¹ を直接計算するのではなく Mᴴ を使うと高速・高精度です。 展開すると： b_mm = S_mm × a_mm = (M × S_se × Mᴴ) × (M × a_se) = M × b_se = M × S_se × Mᴴ × a_mm ∴ S_mm = M × S_se × Mᴴ

具体的に 4 ポート（差動ペア 1 = ポート 1, 2; 差動ペア 2 = ポート 3, 4）の場合、 各要素の展開式（RI 形式、基準インピーダンス 50 Ω 共通の場合）：

── 差動‒差動 (Sdd11) ── Sdd11 = ( S11 − S12 − S21 + S22 ) / 2 ── 差動‒差動 (Sdd21) ── Sdd21 = ( S31 − S32 − S41 + S42 ) / 2 ── コモン‒コモン (Scc11) ── Scc11 = ( S11 + S12 + S21 + S22 ) / 2 ── コモン‒コモン (Scc21) ── Scc21 = ( S31 + S32 + S41 + S42 ) / 2 ── 差動→コモン クロス (Scd11) ── Scd11 = ( S11 + S12 − S21 − S22 ) / 2 ── コモン→差動 クロス (Sdc11) ── Sdc11 = ( S11 − S12 + S21 − S22 ) / 2 （S** は入力ポートが +/−、出力ポートが +/− のシングルエンド Sパラメーター）
STRUCTURE

4. Mixed-mode S 行列の構造

4 ポート（差動ペア 1 + 差動ペア 2）

変換後の 4×4 Mixed-mode S 行列を、 差動 (D) / コモン (C) のブロックに分けて表示すると以下のようになります。 行は「出力モード」、列は「入力モード」を表します。

S_mm =
Sdd11Sdd12Sdc11Sdc12 Sdd21Sdd22Sdc21Sdc22 Scd11Scd12Scc11Scc12 Scd21Scd22Scc21Scc22
← ポート順: D1, D2, C1, C2
（行 = 出力モード、列 = 入力モード）

各ブロックの意味

ブロック記号意味理想的な値
Sdd (差動→差動) Sddij 差動入力に対する差動出力。通常の差動回路特性 設計値どおり
Scc (コモン→コモン) Sccij コモン入力に対するコモン出力。コモンモード伝達特性 差動回路では低い（減衰大）
Scd (差動→コモン) Scdij 差動入力がコモンモードに変換される割合（Mode conversion） 0（変換なし）が理想
Sdc (コモン→差動) Sdcij コモン入力が差動モードに変換される割合（Mode conversion） 0（変換なし）が理想

CMRR（コモンモード除去比）は差動回路の重要指標です：

CMRR [dB] = 20 × log10( |Sdd21| / |Scd21| ) 差動→差動の伝達に対して、コモン→差動の変換（=ノイズ混入）がどれだけ小さいかを示す。 値が大きいほどコモンモードノイズに強い（理想は∞ dB）。

ICD（Insertion loss, Common-mode to Differential）も同様の概念：

ICD [dB] = 20 × log10( |Sdc21| )
IMPEDANCE

5. 基準インピーダンスの扱い

Mixed-mode 変換は基準インピーダンスとセットで考える必要があります。 シングルエンドの基準インピーダンスを Z₀（通常 50 Ω）とすると：

モード基準インピーダンス理由
差動モード Z_diff = 2 × Z₀ = 100 Ω 2 本の線路が直列になるため、シングルエンドの 2 倍になる
コモンモード Z_comm = Z₀ / 2 = 25 Ω 2 本の線路が並列になるため、シングルエンドの 1/2 になる
⚠ 上記の関係は シングルエンドの全ポートが同じ Z₀ の場合 に成立します。 ポートごとに異なる基準インピーダンスが設定されている場合は、 単純な 2 倍・1/2 の関係にはなりません。 Touchstone 2.0 の [Reference] でポートごとに異なる値が指定されている場合、 変換前に基準インピーダンスの正規化が必要です。
Z₀ = 50 Ω の場合： Z_diff = 2 × 50 = 100 Ω （差動ポートの基準インピーダンス） Z_comm = 50 / 2 = 25 Ω （コモンポートの基準インピーダンス） 50 Ω 系で差動インピーダンス 100 Ω を持つ伝送線路は、 Mixed-mode では |Sdd11| ≈ 0 となる（完全整合）。

ポートごとに Z₀ が異なる場合の変換

ポート p の基準インピーダンスを Zp、ポート n の基準インピーダンスを Zn とする場合、 正規化波の変換は次のようになります（Bockelman-Eisenstadt 拡張）：

a_D = √(Zn/(Zp+Zn)) × a_p − √(Zp/(Zp+Zn)) × a_n a_C = √(Zp/(Zp+Zn)) × a_p + √(Zn/(Zp+Zn)) × a_n Zp = Zn = Z₀ の場合は √(1/2) = 1/√2 に帰着する。
HYBRID

6. シングルエンドポートとの混在

全ポートが差動ペアである必要はありません。 一部のポートがシングルエンド（不平衡）のままで、一部だけ差動ペアに変換する "Mixed" 構成（Mixed-mode の名前の由来のひとつ）が可能です。

例: 3 ペア (6 ポート) のうち 2 ペアを差動変換、1 ポートをシングルエンドのまま残す場合 SE (P3) ────────────────── SE + (P1) ─┐ ├── ペア 1 変換 ──── D1, C1 − (P2) ─┘ + (P4) ─┐ ├── ペア 2 変換 ──── D2, C2 − (P5) ─┘ SE (P6) ────────────────── SE 結果: 6×6 行列 ポート順 (Mixed-mode): SE_P3, D1, C1, D2, C2, SE_P6

シングルエンドポートに対応する変換行列 M の対角要素は 1 （変換なし）となり、 他の行・列要素は 0 です。 実装では「変換対象外ポート」として扱い、その行列行・列をそのままコピーします。

TOUCHSTONE 2.0

7. Touchstone 2.0 における Mixed-mode の記述方法

Touchstone 2.0 (IPC-2341A) では [Mixed-Mode Order] キーワードで どのポートが差動ペアかを宣言します。ファイルのデータは 変換後の Mixed-mode 行列 として格納されます。

[Mixed-Mode Order] の書式

[Mixed-Mode Order] <モード1> <シングルエンドポート番号 or ペア定義> ... 記号の意味： D<n> = 差動ペア番号 n の差動ポート C<n> = 差動ペア番号 n のコモンポート S<n> = シングルエンドポート n（変換なし）

具体例：4 ポート全差動

[Version] 2.0 [Number of Ports] 4 [Mixed-Mode Order] D1 C1 D2 C2 ! ポート1=D1（差動ペア1, 差動）, ポート2=C1（差動ペア1, コモン）, ... [Reference] 100 25 100 25 ← Z_diff=100Ω, Z_comm=25Ω, 同上 [Network Data] ! freq Sdd11 Sdc11 Sdd12 Sdc12 ! Scd11 Scc11 Scd12 Scc12 ! Sdd21 Sdc21 Sdd22 Sdc22 ! Scd21 Scc21 Scd22 Scc22 1.0e9 ... [End]

具体例：シングルエンドポート混在

[Version] 2.0 [Number of Ports] 5 [Mixed-Mode Order] S1 D1 C1 D2 C2 ! ポート1=シングルエンド(SE), ポート2=D1, ポート3=C1, ポート4=D2, ポート5=C2 [Network Data] ! 5×5 Mixed-mode S 行列 ... [End]
ℹ Touchstone 2.0 で [Mixed-Mode Order] が宣言されている場合、 [Network Data] に格納されているデータはすでに Mixed-mode 変換済みです。 シングルエンドに戻すには逆変換（S_se = M⁻¹ × S_mm × M）が必要です。
PRACTICE

8. 実用上の注意点

ペア割り当てとポート番号の対応

Touchstone 1.x ファイルには Mixed-mode に関する情報は含まれていません。 どのポートが差動ペアを形成するかは外部から指定する必要があります。 本ツール（Touchstone Mixed-Mode Converter）では、ユーザーが UI 上で ポートのペアを指定することで変換を行います。

正/負の割り当て

差動ペアでどちらが「正側」でどちらが「負側」かを間違えると、 Sdd の位相が 180° ずれます。 物理的な意味（コネクタのピン番号、基板のシンボル表記など）と一致させてください。

⚠ 正/負を逆に割り当てた場合、Sdd11 の位相が反転します（実部と虚部の符号が逆）。 絶対値（|Sdd11|）は変わらないため気づきにくいバグです。 特に Sdd21 の位相特性や群遅延を見るときに影響が現れます。

2 ポートファイルの特殊ケース（差動 1 ポート）

.s2p ファイルの 2 つのポートを 1 組の差動ペアと見なすと、 変換結果は 2×2 の Mixed-mode 行列（Sdd11, Scc11, Scd11, Sdc11）になります。 これは差動コンポーネント（インダクタ、コンデンサ等）の評価によく使われます。

2 ポート → 差動 1 ポートの変換（Z₀=50Ω） Sdd11 = ( S11 − S12 − S21 + S22 ) / 2 Scc11 = ( S11 + S12 + S21 + S22 ) / 2 Scd11 = ( S11 + S12 − S21 − S22 ) / 2 Sdc11 = ( S11 − S12 + S21 − S22 ) / 2 S21 = S12 の場合（相反回路）：Scd11 = Sdc11

奇数ポートへの対応

差動ペアは 2 ポートで 1 組なので、ペア数 × 2 がポート数を超えることはできません。 余ったポート（差動ペアに割り当てられないポート）はシングルエンドとして扱います。 例: 3 ポートの場合 → 最大 1 組の差動ペア + 1 シングルエンドポート。

注意が必要な計算上の落とし穴

  • DB 形式の直接演算： Mixed-mode 変換は複素数の線形演算です。 DB / MA 形式のデータを直接加減算してはいけません。必ず RI（実部・虚部）に変換してから演算し、変換後に DB / MA に戻します。
  • 行列の対称性： 相反回路（パッシブ）なら S_se は対称行列 (Sij = Sji) ですが、 S_mm は一般に非対称です（Scd ≠ Sdc）。
  • 浮動小数点誤差： ほぼ等しい数の差を取る演算（差動モードの計算）では桁落ちが生じることがあります。 特に非常にバランスの取れた差動線路では Scd, Sdc が数値誤差レベルになり得ます。
