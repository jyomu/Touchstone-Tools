1. Touchstone 1.x 規格

Touchstone 1.x（バージョン 1.0 / 1.1）はネットワークアナライザーベンダーが 1980 年代に策定した 事実上の標準フォーマットです。IPC-2341 として後に標準化されましたが、 バージョン 1.x はその前身にあたる非公式仕様です。 拡張子は .s1p〜.sNp (N はポート数)。

オプション行 (#)

ファイルに必ず 1 行だけ存在する設定行。# で始まり、大文字小文字を区別しません。 各トークンはスペース／タブ区切りで、順序は任意です。

# <周波数単位> <パラメーター種別> <データ形式> R <基準インピーダンス> # GHz S RI R 50 ← 最も一般的な組み合わせ # MHz S MA R 50 # Hz Y DB R 75
トークン選択肢説明
周波数単位 Hz kHz MHz GHz 省略時は GHz。大文字小文字不問
パラメーター種別 S Y Z H G Sパラメーター / アドミタンス / インピーダンス / Hパラメーター / Gパラメーター
データ形式 RI MA DB 実部・虚部 / 絶対値・位相[deg] / dB値・位相[deg]
R <値> 任意の正実数 基準インピーダンス [Ω]。省略時は R 50

コメント行 (!)

! 以降の文字（行末まで）はコメントとして無視されます。 行の途中に置くことも可能（インラインコメント）。

! これはコメント行です # GHz S RI R 50 ! オプション行のインラインコメントも有効 1.0 0.99 0.01 0.02 0.03 0.02 0.03 0.98 0.01 ! データ行コメント

データ行の順序

データは周波数昇順で並べます。改行・空白は自由で、複数行にまたがっても構いません。 ただしポート数によってデータの並び順が異なる点が注意事項です（歴史的経緯）。

1 ポート (.s1p)

! freq S11(re) S11(im) 1.0e9 0.9 0.1

2 ポート (.s2p) — 列優先 (Column-major)

2 ポートだけは 列優先（S11, S21, S12, S22 の順）で格納されます。 行列で表すと「第 1 列を先に書き、次に第 2 列」という順序です。

! freq S11 S21 S12 S22 1.0e9 0.9 0.1 0.1 0.05 0.1 0.05 0.8 0.2 ! 行列で表現すると： ! [ S11 S12 ] → 格納順: S11, S21, S12, S22 (列優先) ! [ S21 S22 ]
⚠ 2 ポートだけが列優先です。3 ポート以上は行優先になります。この非対称な仕様は歴史的な設計ミスとして知られており、Touchstone 2.0 で修正されました。

3 ポート以上 (.s3p〜) — 行優先 (Row-major)

N≥3 のファイルでは 1 周波数点ごとに N 行で記述するのが慣例です （ただし改行は任意なので実装はデータ数で判断します）。

! 3-port の例 ! freq S11 S12 S13 ! S21 S22 S23 ! S31 S32 S33 1.0e9 0.8 0.1 0.05 0.02 0.04 0.01 0.05 0.02 0.7 0.1 0.03 0.01 0.04 0.01 0.03 0.01 0.75 0.08

省略時のデフォルト

項目デフォルト値
周波数単位GHz
パラメーター種別S
データ形式MA
基準インピーダンス50 Ω
ℹ オプション行が存在しない場合は上記デフォルトで解釈する実装が多いですが、 厳密には仕様違反です。実用上は # GHz S RI R 50 を明示することを推奨します。

2. Touchstone 2.0 規格

Touchstone 2.0 は IPC-2341A（2008年）として正式標準化されたフォーマットです。 ブラケット [keyword] 形式のメタデータ、ポートごとの基準インピーダンス、 Mixed-mode 記述などが追加されています。拡張子は 1.x と同じ .sNp ですが、 ファイル先頭の [Version] 2.0 で識別します。

ブラケットキーワード一覧

キーワード必須説明
[Version] 2.0必須バージョン識別子。ファイル最初のキーワードであること
[Number of Ports] N必須ポート数
[Options]任意1.x と同じ書式のオプション行をここに記述
[Number of Frequencies] N任意周波数点数（事前宣言）
[Number of Noise Frequencies] N任意ノイズデータ周波数点数
[Reference]任意ポートごとの基準インピーダンス（スペース区切りで N 個）
[Matrix Format] Full|Lower|Upper任意行列の格納形式（省略時は Full）
[Mixed-Mode Order]任意Mixed-mode ポートの定義（後述）
[Two-Port Order] 12_21|21_12任意2 ポートの並び順（後述）
[Begin Information] / [End Information]任意任意のテキストメタデータブロック
[Network Data]必須ネットワークデータ開始マーカー
[Noise Data]任意ノイズデータ開始マーカー
[End]必須ファイル終端マーカー

データセクション

[Network Data] 以降に、周波数ごとのデータを行列行優先で記述します。 2 ポートを含むすべてのポート数で 行優先 です（1.x の列優先例外なし）。

[Version] 2.0 [Number of Ports] 2 [Reference] 50 50 [Network Data] ! freq S11 S12 S21 S22 1.0e9 0.9 0.1 0.1 0.05 0.1 0.05 0.8 0.2 [End]

2 ポートの互換モード ([Two-Port Order])

デフォルトは行優先 (12_21: S11, S12, S21, S22) ですが、 [Two-Port Order] 21_12 を指定すると Touchstone 1.x 互換の列優先 (S11, S21, S12, S22) として解釈されます。

[Two-Port Order] 21_12 ← 1.x 互換モード（列優先） [Two-Port Order] 12_21 ← 2.0 デフォルト（行優先）
ℹ [Two-Port Order] キーワードが存在する場合は 2.0 ファイルです。 存在しない場合のデフォルトは 12_21（行優先）です。

3. Keysight 独自 ! ヘッダー

Keysight（旧 Agilent / HP）のネットワークアナライザー（ENA シリーズ、PNA シリーズ等）は Touchstone 1.x ファイルに独自の ! コメント行を挿入します。 これらは仕様上は単なるコメントですが、多くのソフトウェアがこれを解釈して ポートインピーダンスや計測条件を取得します。

キーワード一覧

キーワード行内容例
! Keysight Technologies,<モデル>,<シリアル>,<ファーム> 機器識別（メーカー・型番・S/N・FW） ! Keysight Technologies,E5063A,MY12345678,A.15.31
! Agilent Technologies,<モデル>,...（旧称） 旧 Agilent 時代の機器識別 ! Agilent Technologies,E5071C,MY09876543,A.09.80
! Date: <日時> 計測日時 ! Date: Wednesday, May 06, 2026 09:36:11
! Touchstone version: <バージョン> 出力した Touchstone バージョン ! Touchstone version: 1.1
! Port Z0: <r1> <r2> ... ポートごとの基準インピーダンス [Ω] ! Port Z0: 50.000 50.000
! Port Impedance: <r1> ... 同上（別表記） ! Port Impedance: 50.000 50.000
! Port Name: <名前 1> <名前 2> ... ポートのラベル名 ! Port Name: Port1 Port2
! Reference: <条件> キャリブレーション参照情報 ! Reference: Full 2-Port
! Start: / ! Stop: / ! Step: 計測周波数レンジ設定 ! Start: 300000 Hz
! Points: <数> 周波数点数 ! Points: 1601
! Power: <値> <単位> 入力電力レベル ! Power: -10 dBm
! IFBW: <値> <単位> IF バンド幅 ! IFBW: 1000 Hz
! Averaging: <オン/オフ> [Factor: <数>] アベレージング設定 ! Averaging: ON Factor: 16

IFBW: 1000 Hz ! Averaging: OFF ! Port Z0: 50.000 50.000 # Hz S RI R 50 300000 9.9898e-01 6.2383e-03 4.0625e-07 4.3651e-07 ... ...

4. バージョン × Keysight ヘッダーの組み合わせと挙動

実際のファイルは「Touchstone のバージョン」と「Keysight ヘッダーの有無・内容」の組み合わせで 多様なケースが存在します。以下に主要なパターンをまとめます。

Touchstone 1.x + Keysight ヘッダーあり
  • ◎ 最多 Keysight 製品の標準出力
  • ! コメントは仕様上無視してよい
  • ! Port Z0: に非 50 Ω 値があれば考慮が必要
  • ポート名・計測条件をメタデータとして活用可能
Touchstone 1.x + コメントなし
  • ◎ 汎用 シミュレーターなどの出力
  • オプション行だけを解釈すればよい
  • 基準インピーダンスは R 値のみ参照
Touchstone 2.0 + Keysight ヘッダーあり
  • △ 少数 新しい Keysight 機器が出力
  • [Begin Information]...[End Information] にも同等情報が入ることがある
  • ! コメントは互換情報として残る場合がある
  • ポートインピーダンスは [Reference] を優先する
Touchstone 2.0 + コメントなし
  • ◎ 標準 シミュレーター・EDA ツールの出力
  • [Reference] でポートごとのインピーダンスを指定
  • [Mixed-Mode Order] で Mixed-mode が明示される

基準インピーダンスの優先順位

同一ファイルに複数の基準インピーダンス情報がある場合の推奨優先順位：

  1. Touchstone 2.0 [Reference] キーワード — 明示的なポートごとの値、最優先
  2. Keysight ! Port Z0: / ! Port Impedance: — ベンダー拡張、ポートごとに指定可能
  3. オプション行 R <値> — 全ポート共通のフォールバック
  4. デフォルト 50 Ω — 何も指定がない場合

2 ポートデータ並び順の判定フロー

ファイル読み込み時 ↓ [Version] 2.0 が存在する？ ├─ Yes → Touchstone 2.0 │ ↓ [Two-Port Order] が存在する？ │ ├─ 21_12 → 列優先 (1.x 互換) で解析 │ ├─ 12_21 → 行優先 (2.0 標準) で解析 │ └─ なし → 行優先 (2.0 標準) で解析 └─ No → Touchstone 1.x ↓ ポート数は？ ├─ 2 ポート → 列優先 (S11, S21, S12, S22) で解析 └─ その他 → 行優先 (S11, S12, ..., SNN) で解析

Keysight ヘッダーと [Begin Information] の対応

5. 実装上の注意点

パーサー実装時のチェックリスト

  • 大文字小文字を区別せずトークンを解釈する（オプション行、ブラケットキーワード）
  • データ行のインラインコメント（! 以降）を除去してから数値解析する
  • 複数行にまたがるデータを連結してから周波数ごとに分割する（改行は無意味）
  • 2 ポートファイルでバージョンを確認してから列優先 / 行優先を決定する
  • ! Port Z0: を見つけた場合は R 値より優先するかどうかをユーザーに委ねる、または最優先で扱う
  • Touchstone 2.0 の [Matrix Format] Lower / Upper は実装が必要（下三角 / 上三角のみ格納）
  • ! Keysight Technologies, と ! Agilent Technologies, の両方を考慮する

よくある罠

✕ 2 ポートの列優先を忘れる： 多くのパーサーが「S11, S12, S21, S22」（行優先）と誤実装しています。 Touchstone 1.x の 2 ポートは「S11, S21, S12, S22」（列優先）が正しい解釈です。
✕ DB 形式の 0 マグニチュード： -999.999 dB や -inf dB はゼロを意味する慣例値です。 変換時に 0 V として扱わないと誤差が生じます。
⚠ オプション行が複数存在するファイル： 仕様では 1 行のみとされていますが、一部ツールが複数出力する場合があります。 最初に見つかった行を採用するのが一般的な実装です。
